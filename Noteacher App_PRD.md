# AI学习档案馆 App | PRD 1.1 （识卷优先版）

> 面向C端学生与家长的AI学习档案系统，以“上传一张作业→AI自动批阅→错题库→学习档案”为核心闭环。  
> 本版本在原PRD基础上强化了“识卷优先 / Agent-First”战略。

## 1️⃣ 产品定位与愿景

| 项目       | 内容                                   |
| -------- | ------------------------------------ |
| **产品名称** | AI学习档案馆（AI Learning Archive）         |
| **产品形态** | 移动App + 家长Web端                       |
| **面向用户** | 中小学学生（6–16岁）与其家长                     |
| **核心使命** | 让学生的每一份作业都成为成长的证据。                   |
| **愿景**   | 构建学生的AI学习档案馆，让AI陪伴他们完成自我反思、纠错、计划与成长。 |
| **产品口号** | “上传一张作业，看见自己的成长。”                    |

---

## 2️⃣ 用户画像与核心痛点

### 👩‍🎓 学生端用户画像
- 年龄：10–16岁  
- 使用场景：日常作业、阶段测验  
- 学习习惯：碎片、缺乏系统复盘  
- 痛点：
  1. 做错题后没有系统记录，重复犯错；  
  2. 缺乏“AI老师”即时反馈；  
  3. 不知道如何从错误中改进。

### 👩‍👦 家长端用户画像
- 年龄：30–45岁  
- 角色：学习陪伴者与监督者  
- 痛点：
  1. 无法理解孩子错题分布与趋势；  
  2. 看不到长期成长轨迹；  
  3. 不希望反复催学，想要数据化陪伴。

---

## 3️⃣ 产品目标与核心价值

| 层级 | 目标 | 关键价值 |
| ---- | ---- | ---- |
| **短期目标** | 完成“图片 → 高置信结构化识卷 JSON” | 保证识卷准确率，为后续模块提供可信输入 |
| **中期目标** | 在识卷结果上实现“判题+错因+讲解” | 形成完整AI批阅闭环 |
| **长期目标** | 建立“学生学习档案+成长曲线” | 长期学习追踪与个性化计划 |

> 💡 **识卷准确是本产品的第一性原理。** 判题、错因、讲解都依赖识卷的正确性。

---

## 4️⃣ 成功指标（KPI）

### 📊 识卷层指标
- 版面检测 F1 ≥ 0.95  
- 印刷体 WER ≤ 2%  
- 手写体 CER ≤ 6%  
- 公式等价率 ≥ 97%  
- 致命错误率 ≤ 0.5%  
- 低置信度率 10–20%  

### 🧠 批阅层指标
- 判题一致率 ≥ 90%（对照人工批改）  
- 错因分析合理率 ≥ 85%  
- 思路讲解可理解率 ≥ 90%  

### 📈 用户层指标
- 上传→批阅完成率 ≥ 90%  
- 家长端留存率 ≥ 70%（30天）  
- 用户满意度 ≥ 8.0/10

## 5️⃣ 功能模块与流程图

### 【总体流程】
```
学生上传作业/试卷 → AI自动识别与批阅 → 错题库更新 → AI学习计划生成 → 家长查看报告
```

### 【模块树】
| 模块 | 子模块 | 功能 |
| ---- | ---- | ---- |
| 📷 上传模块 | 作业上传 / 试卷上传 | 拍照上传图片、批量上传 |
| 🤖 AI批阅模块 | 题目识别 / 自动判分 / 错因分析 | 多模态Agent自动识别并输出结构化结果 |
| 🧩 错题库模块 | 错题归档 / 查询 / 删除 / 标记掌握 | 按科目+日期自动归档错题 |
| 🧠 学习计划模块 | AI生成复习计划 / 复习进度 | 基于错题集生成个性化计划 |
| 📊 报告模块 | 日报 / 周报 / 阶段报告 | 趋势图与AI总结 |
| 👩‍👦 家长端模块 | 学情看板 / 对比分析 / 档案导出 | 可视化展示孩子学习趋势 |
| ⚙️ 系统模块 | 账号管理 / 数据同步 / 日志 | 隐私与安全 |

### 👩‍🎓 学生端
| 场景 | 动作 | 系统反馈 |

## 7️⃣ 系统架构与AI模块（Agent-First）

```
+----------------------------------------------------+
|                 APP 层（学生/家长端）              |
| Flutter / ReactNative / Vue3                       |
| - 上传界面、错题库、学习报告                      |
| - 通过 HTTPS 调用 API                              |
+----------------------------------------------------+
                ↓（API 调用）
+----------------------------------------------------+
|                 AI Agent 层                        |
| Claude Agent SDK (Python / TS) + FastAPI           |
| - layout/ocr-merge/math-validate/quality-gate       |
| - OCR/公式识别（百度/DeepSeek/Mathpix）            |
| - 输出识卷JSON (RecognitionPayload)               |
+----------------------------------------------------+
                ↓（异步数据）
+----------------------------------------------------+
|                 数据层                             |
| PostgreSQL / MongoDB / MinIO / Redis               |
| - 错题库、用户档案、上传记录                      |
| - 学习画像与模型训练数据集                        |
+----------------------------------------------------+
```

## 9️⃣ MVP范围与阶段目标

| 阶段 | 目标 | 范围 | 验收标准 |
| ---- | ---- | ---- | ---- |
| **Phase 1 – 识卷MVP** | 图片→识卷JSON | 数学（选择+填空） | 识卷KPI全达标 |
| **Phase 2 – 判题/错因/讲解** | 识卷JSON → 批阅 | 与人工一致率≥90% | |
| **Phase 3 – 报告与档案** | 家长端闭环 | 学习报告与错题库 | |

---

## 🔟 接口与数据流

- `POST /api/recognize` → 输入图片，返回识卷JSON  
- `POST /api/grade` → 输入识卷JSON，返回判题JSON  
- `POST /api/report` → 输入学生ID，返回趋势报告

---

## ⚙️ 风险与对策（识卷相关）
- 手写体混淆 → 歧义字典 + 重拍提示  
- 复杂版面 → 拍摄规范 + layout-agent优先  
- OCR偏差 → 多引擎融合 + 置信度投票  
- 误判风险 → 质量门 + 低置信度人工复核  

---

## ✅ 总结
**先让AI学会“看懂作业”，再让App学会“呈现成长”。**  
识卷准确率是整个系统的核心生命线，Agent 与 App 以清晰API解耦，确保能独立迭代与快速集成。
