# Stakeholder Feedback Analysis & Prioritization

## I. 需立即决策 (P0 - Critical for Phase 1: Core Agent)
*这些问题直接影响 Python Agent 的 Prompt 设计、数据结构和核心逻辑，必须现在对其。*

### 1. Agent 核心逻辑 (Agent Logic)
*   **(P1) 严格模式细节**：英语语义相似度阈值如何定义？是否支持关键词“必含”配置？
*   **(P1) 几何与步骤校验**：Agent 如何输出几何图形的判断结果（是只说对错，还是返回坐标）？
*   **(P1/P2) 边界情况处理**：
    *   学科交叉（数学题里的英语单词，英语题里的算式）怎么处理？
    *   部分正确（步骤对答案错）的引导策略？
    *   **引导终止机制**：Partner 2 提到的“多次引导失败”（如 3 次）后是否直接给答案？
*   **(P1) 记忆边界 (Memory)**：确认“当前会话记忆”与“长期画像”的隔离墙。Agent 在本次辅导中是否**绝对不**读取长期画像（只写入）？

### 2. 关键架构决策 (Core Architecture)
*   **(P1) 交互机制**：Node.js BFF 与 Python Agent 到底是 **HTTP 直接调用** 还是 **消息队列 (MQ/Redis)**？
    *   *影响*：系统延时、各种超时重试策略。
*   **(P1) 坐标系标准**：错题切片和全页快照的坐标系（Bounding Box）标准（比如 normalized 0-1 还是 pixel）。

## II. 可暂缓讨论 (P1 - Phase 2: Productization/BFF)
*这些问题主要涉及 App 端体验、运营后台和非核心流程，建议在 Agent 跑通后讨论。*

### 1. 产品与体验 (Product & UX)
*   **角色扩展**：老师角色、复杂的权限管理。 -> *MVP 聚焦家长/学生。*
*   **通知通道**：推送/短信/邮件具体实现。 -> *BFF 实现细节。*
*   **交互细节**：
    *   上传中断/追加照片。
    *   离线/弱网模式。
    *   学习进度即时测试。
    *   数据导出/注销。
*   **人工复核流**：具体的 UI 交互设计（多选确认等）。

### 2. 详细技术实现 (Tech Implementation)
*   **SSE 错误码详表**：先定大框架，具体码表开发时定。
*   **安全加密**：URL 签名等安全措施。
*   **可观测性**：日志/Tracing 方案。

*   **数据存储**：错题切片 + 全页快照（已确认）。
*   **多端一致性**：云端化设计天然支持。
